## 2. 資料庫定義與相依關係

### 2.1 既有 tables（不改動）

| Table | 用途 | V6.1 關聯 |
|---|---|---|
| `historical_prices` | OHLCV 日 K | ML 訓練特徵來源；擴充至 2 年 |
| `analyst_targets` | 分析師目標價 | 萃取為 20D/60D 特徵 |
| `transactions` | 交易紀錄 | 計算 avgCost / avgHoldingDays |
| `holdings` | 持倉記錄 | positionValue / positionPct |
| `daily_news_digest` | 每日新聞摘要 | 未來情緒特徵（V6.1 暫不使用） |
| `fundamental_data` | 基本面資料 | 未來特徵（V6.1 暫不使用） |

### 2.2 新增 `modelpredictions` table

```ts
export const modelPredictions = sqliteTable('modelpredictions', {
  id: integer('id').primaryKey({ autoIncrement: true }),
  symbol: text('symbol').notNull(),
  market: text('market').notNull(),          // 'TW' | 'US'
  modelName: text('model_name').notNull(),    // 'RF_v1', 'LSTM_v2' 等
  horizonDays: integer('horizon_days').notNull(), // 5 | 20 | 60
  runAt: text('run_at').notNull(),            // YYYY-MM-DD (市場時區)
  startDate: text('start_date').notNull(),    // 預測起點日
  endDate: text('end_date').notNull(),        // 預測終點日
  medianPathJson: text('median_path').notNull(), // [{date, price}]
  lowerPathJson: text('lower_path'),          // 信賴區間下界
  upperPathJson: text('upper_path'),          // 信賴區間上界
  metaJson: text('meta_json'),               // featureVersion, trainWindowYears 等
  createdAt: integer('created_at').notNull(),
});
```

**索引**：`(symbol, market, horizonDays, runAt)` — 加速 prediction-history API 查詢。

### 2.3 Table 關聯說明

- `modelPredictions.symbol + market` → join `historicalPrices` 計算誤差（MAE/MAPE）
- `modelPredictions.medianPath.date` 對齊 `historicalPrices.date` 計算實際 vs 預測偏差
- `analystTargets` → `analystConsensusService` 萃取特徵 → `mlPredictionService` 特徵向量
- `transactions + holdings` → `personalAdviceService` 計算持倉狀態 → `PersonalAdvice`

### 2.4 Migration 策略

- 新增 `modelpredictions` 用 `CREATE TABLE IF NOT EXISTS`，不影響既有資料
- 擴充歷史價：僅補最早日期之前的缺口（`INSERT OR IGNORE`），已有日期不覆蓋

## 3. 資料流與衝突風險分析

### 3.1 現有資料流（V6.0）

```
Yahoo/TWSE → historicalPrices (1yr) → Technical 頁 K 線
Excel → analystTargets → Technical overlay (目標價帶狀)
Excel → transactions/holdings → Portfolio 頁
MLPrediction.tsx → simulateRFPrediction() [client-side mock，無真實模型]
```

### 3.2 V6.1 新資料流

```
Yahoo/TWSE → historicalPrices (2yr) → ML 訓練窗口擴大
每次預測觸發：
  historicalPrices → OHLCV 特徵 ─┐
  analystTargets → analyst 特徵 ──┤→ mlPredictionService → 5D/20D/60D
  (future) newsDigest → sentiment ┘
      └→ insertModelPrediction() → modelpredictions 表 [永遠新增，不覆蓋]
      └→ 前端 Prediction 頁顯示預測路徑

/api/prediction-history → 讀 modelpredictions + historicalPrices → 計算誤差
/api/personal-advice → PersonalPositionState + HorizonPredictionSummary + AnalystFeatures → PersonalAdvice
```

### 3.3 風險與衝突清單

| 風險 | 說明 | 緩解策略 |
|---|---|---|
| 模型版本更新覆蓋歷史 | 新模型上線後舊預測資料消失 | `insertModelPrediction` 永遠 INSERT，不 UPDATE；舊 run 以 `metaJson.deprecated=true` 標記 |
| 歷史價補齊誤刪資料 | 補 2 年時不小心清掉已有資料 | 僅 INSERT OR IGNORE，不做 DELETE；補齊前先 count 現有行數做 sanity check |
| analystTargets 重複匯入 | Excel 重匯時重複 | 沿用既有 `ON CONFLICT (symbol, market, institution, analystDate)` UPSERT，不刪除其他日期 |
| MLPrediction 重筆影響現有路由 | routes.ts 改動可能衝突 | 舊 ML endpoint 改名保留，新 endpoint 新命名 |
| 個人建議依賴 holdings 一致性 | holdings 若有空值算法崩潰 | positionPctOfPortfolio / unrealizedPct 設 null safety，不足資料時 primaryAction 回傳 `hold` |

## 4. 歷史價擴充策略

### 採用：2 年（730 天）

**理由：**
- Yahoo Finance 免費 API 支援任意歷史起點，無 API 限制
- SQLite 每股 2 年 ≈ 500 bars × ~80 bytes = 40KB；17 檔美股 ≈ 680KB，可接受
- 3 年在大多數標的上訓練增益遞減，2 年對 RF 模型已足夠
- 若個別 symbol 實際上市未滿 2 年（如 TEM、NTLA），使用可取得的全部歷史，`metaJson.trainWindowYears` 記錄實際年數

### 實作要點

1. 在 `stockService.ts` 修改 `initializeHistoryPool`：`startYear` 從 today - 365 改為 today - 730
2. 已有資料的 symbol：檢查最早日期，若距今 < 700 天 → 補齊前段缺口
3. `routes.ts` 的 `/api/history/refresh-all` 也同步調整為 2 年窗口

## 5. ML 模型規格（V6.1 初版）

### 特徵清單

**技術面（所有 horizon）：**
- `close_pct_5d`, `close_pct_20d` — 近期漲跌幅
- `rsi_14` — RSI
- `vol_ratio_20d` — 成交量比率（今/20日均）
- `atr_14_pct` — ATR / 收盤價（波動率）
- `ma20_dist_pct`, `ma60_dist_pct` — 均線乖離率

**Analyst 特徵（僅 20D / 60D）：**
- `upside_avg_ratio` — (avgTarget - current) / current
- `band_width` — (high - low) / current
- `bullish_ratio` — 樂觀評級佔比
- `avg_score` — 評分均值
- `days_since_last_update` — 距上次更新天數

### 模型選擇

- **5D**：Random Forest（`RF_v1`），快速，無需大量訓練資料
- **20D**：Random Forest + analyst 特徵（`RF_analyst_v1`）
- **60D**：Random Forest + analyst 特徵（`RF_analyst_v1`）（V6.1 先用 RF，未來可換 LSTM）

### 輸出

每個 horizon 輸出：
- `medianPath`：中位預測路徑（逐日）
- `upperPath` / `lowerPath`：信賴區間（±1σ 或分位數）
- `metaJson`：`{ featureVersion, modelName, trainWindowYears, trainSamples, featureList }`

## 6. 個人化建議規則（初版）

| 條件 | primaryAction |
|---|---|
| expectedReturn < -maxLossPct | `cut_loss` |
| unrealizedPct < -maxLossPct | `cut_loss` |
| expectedReturn > 10% AND positionPct < maxSinglePositionPct | `add_on_dip` |
| expectedReturn > 15% AND unrealizedPct > 20% | `take_profit_partial` |
| analyst upsideAvg < -5% | `avoid_new_entry` |
| 其他 | `hold` |

預設 StrategyProfile：`maxSinglePositionPct=20%`, `maxLossPctPerPosition=8%`, `horizonPreference='swing'`（20D）
